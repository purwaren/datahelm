# DataHelm Spike Evidence

This folder is reserved for `M0` spike outputs.

Recommended structure:

- `reports/`: spike reports such as `sp-01-report.md`
- `payloads/`: normalized example payloads for metadata, results, and capability maps
- `notes/`: short implementation notes, caveats, and unresolved questions

Guidelines:

- prefer short evidence artifacts over long narrative notes
- keep secrets and live credentials out of the repository
- record engine-specific caveats explicitly
